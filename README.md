## ❓ FAQ
> [!TIP]
> Kick’s API is **rate-limited**. Avoid monitoring >50 channels without a proxy.

**Q: Does this work with private streams?**
No. The tool only accesses **publicly available** stream data (same as the Kick website).

**Q: Can I use this to boost my own stream’s viewer count?**
**No.** Kick detects and bans **artificial viewbots**. This tool is for **analytics only**.

**Q: Why does it need administrator privileges?**
To **bypass Windows’ network stack limits** and ensure low-latency monitoring. No system files are modified.

**Q: Is this detectable by Kick?**
The tool uses **the same API calls as Kick’s frontend**. No abnormal traffic patterns are generated.

**Q: Can I run multiple instances?**
Yes, but **each instance must use a different proxy** to avoid rate limits.
